## ✅ Nouveautés fonctionnelles

### 🏢 Employeurs

#### Rattachement automatique à l'OPCO
- **Recherche automatique** : Lorsque vous créez un employeur, le système recherche automatiquement l'OPCO (Opérateur de Compétences) correspondant selon :
  - L'**IDCC** (Identifiant de Convention Collective) renseigné
  - Le **SIRET** de l'entreprise
- **Affichage dans le détail** : Consultez l'OPCO rattaché directement depuis la fiche employeur avec une dialog dédiée
- **Modification possible** : Vous pouvez modifier l'OPCO rattaché si nécessaire
- **Protection par droits** : L'accès en lecture et modification de l'OPCO est contrôlé par les permissions
- **Utilisation dans les contrats** : L'OPCO de l'employeur est automatiquement alimenté par défaut lors de la création d'un contrat

#### Paramétrage des statuts personnalisés
- **Créer des statuts** : Définissez vos propres statuts pour qualifier vos employeurs
- **Afficher et modifier** : Gérez vos statuts personnalisés depuis le paramétrage
- **Supprimer** : Supprimez les statuts devenus obsolètes
- **Utilisation** : Appliquez ces statuts personnalisés sur vos fiches employeurs

### 📋 Dossiers de formation

#### Gestion de la période SFP avant contrat

**Annulation d'une SFP**
- **Annuler une SFP avant contrat** : Vous pouvez désormais annuler une période de mise en situation professionnelle (SFP) avant la signature du contrat
- **Persistance garantie** : L'annulation est correctement enregistrée dans le système

**État "SFP terminée"**
- **Nouvel état** : Un apprenant peut avoir une SFP dans l'état "Terminée"
- **Gestion automatique** : Le système gère automatiquement les transitions d'états de la SFP
- **Date de fin théorique** : La date de fin théorique de la SFP avant contrat est calculée à **3 mois moins 1 jour** selon les règles métier

#### Arrêt de formation
- **Nouveau geste métier** : Vous pouvez déclarer l'**arrêt de formation** d'un apprenant
- **Cas d'usage** : À utiliser lorsqu'un apprenant s'est présenté aux cours mais ne poursuit plus sa formation
- **Distinction** : Différent de l'annulation, l'arrêt signifie que la formation a effectivement débuté

### 📄 Contrats

#### Annulation de contrat non commencé
- **Annuler un contrat** : Vous pouvez annuler un contrat qui n'a pas encore commencé
- **Gestion des cas** : Utile pour les contrats en attente qui ne se concrétiseront finalement pas

#### Employeur connu
- **Tag "Employeur connu"** : Dans la recherche d'employeurs lors de la création d'un brouillon de contrat, les employeurs déjà enregistrés dans votre base sont identifiés par un tag
- **Facilite la sélection** : Repérez rapidement si l'employeur existe déjà ou s'il faut le créer

#### Réorganisation selon la version de contrat
- **Parcours apprenant déplacé** : La section "Parcours apprenant" est déplacée dans l'interface du contrat car son emplacement dépend de la version de contrat
- **Type employeur déplacé** : Le type d'employeur est également repositionné car il varie selon la version de contrat
- **Amélioration de la cohérence** : Ces déplacements permettent d'adapter l'interface aux spécificités de chaque version

#### Changement de version amélioré
- **Gestion des données référentiel** : Le changement de version de contrat gère correctement les données du référentiel
- **Corrections** : Résolution de bugs lors du changement de version

### 🎓 Sessions de formation

#### Choix de la voie d'accès à l'inscription
- **Sélection à l'inscription** : Lors de l'inscription d'un apprenant à une session de formation, vous pouvez choisir la **voie d'accès** utilisée
- **Flexibilité** : Permet de préciser le contexte d'inscription (apprentissage, contrat de professionnalisation, etc.)

### ⚙️ Paramétrage et référentiel

#### Type de diplôme avec version de contrat
- **Version de contrat associée** : Chaque type de diplôme peut maintenant être associé à une version spécifique de contrat
- **Recherche exposée** : La version de contrat est disponible dans les résultats de recherche de type diplôme
- **Adaptation automatique** : Le système utilise cette information pour adapter le contrat selon le type de diplôme choisi

#### Mapping du libellé affiché
- **Correction** : Le mapping du libellé affiché des types de diplôme est corrigé pour afficher correctement les libellés personnalisés

### 📝 Normalisation des données

#### Formatage automatique des noms et prénoms
- **Règles communes** : Mise en commun du format des noms et prénoms dans toute l'application
- **Formatage avant envoi** : Les noms et prénoms sont automatiquement formatés avant l'envoi au serveur
- **Règles spécifiques** : Application de règles de formatage adaptées selon le contexte
- **Cohérence** : Garantit une cohérence des données dans l'ensemble du système

## 🛠️ Correctifs et fiabilisation

### Employeurs
- **Modification en erreur** : Correction du bug lors de la modification d'un employeur

### Formations
- **Refus en approbation** : Correction du processus de refus d'une formation en cours d'approbation. Le refus d'ue formation en approbation ne se réalisait pas. A présent, Lorsque l'on refuse une formation en approbation, les actions de formations liées se refusent également.

### Machines
- **Protection rôles et groupes** : Les rôles et groupes non assignables sont correctement protégés. Il n'est plus possible d'assigner les rôles administrés par la plateforme sur une machine.
- **Historique** : Ajout de l'historique des modifications sur les machines

### Documents
- **Type de document** : Le type de document du template est correctement transmis lors de la génération

### Permissions
- **Lecture systématique** : Correction des permissions pour ajouter la lecture automatiquement dans certains contextes
- **Mappings templates** : Corrections des erreurs de mapping dans les templates. La catégorie de l'employeur est bien géré dans le CERFA aiansi que le mode contractuel.

- **Recherche OPCO** : Ajout de la recherche d'OPCO dans le référentiel
